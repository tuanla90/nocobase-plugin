# @tuanla90 plugin registry

Catalog of the `@tuanla90` NocoBase plugins in this workspace.
Source: `packages/@tuanla90/*`. Packaged tgz: `latest/@tuanla90/` (current) + `archive/@tuanla90/` (history).
**33 active plugins.** Last updated: 2026-07-20.

> **Bilingual UI i18n — DONE for all plugins (2026-07-15).** Every plugin's *internal* UI strings (not just the manifest `displayName`) now route through NocoBase i18n and switch en/vi. Scheme by source language: VN-source plugins use the Vietnamese string as the i18n key + `en-US.json` (safe — nb i18next has no `fallbackLng`, so a vi-VN miss returns the key; see `docs/SHARED-DEDUP-AUDIT.md` sibling note & the build-guide §R1); English-source plugins (change-log, global-search, status-flow, custom-icons, pwa) use English keys + `vi-VN.json`. All 12 touched this session built, deployed to `nb-local`, and verified live (served bundle `addResources` + inlined locale). **Known remaining gap:** `@tuanla90/shared/fieldPicker.tsx` hardcodes `＋ Chèn cột` / `Chọn field…` in Vietnamese → leaks VN into every plugin's field picker even for en users; fix belongs in shared.

## Build & packaging

- Recipes: `build-env/recipes/run-<name>-build.sh` → `nocobase-build --tar` (per-plugin stubs/quirks live in each recipe).
- Deliverable `.tgz` → `latest/@tuanla90/` (one current version each) + `archive/@tuanla90/` (all versions).
- As of 2026-07-18, `latest/@tuanla90/` holds all 29 and each is installed + enabled on nb-local. **`ip-guard` (0.2.0) installed + enabled 2026-07-18** (tables `ptdlIpAccessConfigs`+`ptdlIpAccessLogs` created, applicationPlugins enabled=1; loads clean) — firewall behaviour (block/allow/monitor/proxy + app-lane ordering) not yet functionally exercised.
- Per-plugin details, setup and limits: each plugin's `README.md` where present (e.g. `gsheet-sync`, `global-search`, `ai-column`, `login-lite`, `custom-header`).
- **`nb-cloner` (1.9.1) added 2026-07-19** — re-scoped from an older `@my-project/plugin-nb-cloner` 1.7.3 (RAR import) and standardized: bilingual i18n (`src/locale`, English-source keys), `@tuanla90/shared` (ConfigContainer + formatNumber), dual-lane (classic + /v/), a PostgreSQL dialect guard, README + README.vi-VN. Built (`recipes/run-nb-cloner-build.sh`) + tgz in `latest/@tuanla90/` + `archive/@tuanla90/`. **1.9.0:** collection classifier — the export list groups every collection as **user / plugin / system / deleted** (managed-in-`collections`-table + runtime `origin` + a `ptdl*` name heuristic + physical-table check) with a category filter defaulting to *My collections*, so a fresh app shows the ~16 real tables instead of ~105; also **removed the self-update tab/endpoint** (redundant with Plugin Manager + RCE risk). **1.9.1:** goal presets on the Export tab — **📦 Backup** (all your tables + data), **🧬 Clone** (structure/UI, no data), **↺ Default** (reset) — one click sets the toggles + selection. **1.9.2:** fixed *My collections = 0* — the "user" test wrongly required runtime `origin === 'core'`, but the data-source loader stamps user collections with its own origin, so all real tables fell into *System*; now user = managed (`collections` DB row) + not a `SYSTEM_COLLECTION_NAMES` name, and the managed set is read via reliable raw SQL. **Deployed + enabled on nb-local** (sqlite); classifier verified against the DB (user=16, roles/users→system). Export/import still need a **PostgreSQL** app to exercise (dialect guard blocks sqlite). Ships to `/v/` + `/admin` as a **Settings → NB Cloner** page (`ptdl-nb-cloner`).
